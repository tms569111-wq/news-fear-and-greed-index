# 면접 방어 포인트

## 프로젝트 한 줄 설명

기존 Python 기반 뉴스 감성 분석 연구를 Spring Boot 웹서비스로 확장하여, 종목별 뉴스 공포·탐욕 지수와 기사별 감성 결과를 웹에서 조회할 수 있도록 구현한 프로젝트입니다.

## 왜 Python과 Spring Boot를 나눴나?

Python은 KoNLPy/Okt, pandas, 크롤링 라이브러리 생태계가 좋아 뉴스 수집과 감성 분석에 적합합니다.  
Spring Boot는 웹 요청 처리, DB 조회, 계층형 구조, 운영형 백엔드 구현에 적합합니다.  
따라서 Python은 분석 엔진, Spring Boot는 서비스 제공 계층으로 역할을 분리했습니다.

## 왜 DB 공유 방식을 선택했나?

사용자 요청 때마다 뉴스 크롤링과 형태소 분석을 수행하면 응답 시간이 길고 외부 사이트 상태에 의존하게 됩니다.  
그래서 Python 배치가 분석 결과를 MySQL에 미리 저장하고, Spring Boot는 저장된 결과를 빠르게 조회하도록 설계했습니다.

## 왜 3개 테이블로 시작했나?

처음부터 과도하게 복잡한 DB를 만들면 구현이 지연됩니다.  
그래서 분석 대상은 `assets`, 기사 원본과 기사별 결과는 `news_articles`, 날짜별 평균 지수는 `daily_news_index`로 분리해 MVP를 만들었습니다.

## 왜 daily_news_index를 따로 만들었나?

날짜별 차트를 그릴 때 매번 기사 전체를 다시 계산하면 비효율적입니다.  
미리 계산된 날짜별 평균 지수를 별도 테이블에 저장하면 조회 성능이 좋아지고, Spring Boot는 단순 조회에 집중할 수 있습니다.

## 이후 개선 방향

- 실제 Python 크롤러와 DB 적재 자동화
- 주가 데이터와 RSI/MACD 비교 테이블 추가
- 배치 실행 이력 저장
- EC2 배포
- Docker Compose 적용
- 예외 처리와 로그 개선
